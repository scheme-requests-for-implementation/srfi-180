Abstract

This library describes a JavaScript Object Notation (JSON) parser and printer. It supports streaming parses of JSON that may be bigger than memory.

Rationale

JSON is a de facto industry standard for data exchange.

For best interoperability, the sample implementation is based on RFC7159, and the tests are based on JSONTestSuite.

The mapping between JSON types and Scheme objects is not trivial because a given mapping might not be the best for every situation. That is one of the reasons that this library exposes a stream reader, which allows users to specify the Scheme representation and have efficient code.

Specification

(json-error? obj) → boolean

Returns #t is OBJ is an error object. This exception can be raised during json-stream-read, json-read or json-write.

(json-error-reason obj) → string

Return a string explaining the reason for the error. This should be human-readable.

(json-null? obj) → boolean

Return #t if OBJ is the scheme symbol 'null, which represents the JSON null in Scheme. In all other cases, return #f.

(json-stream-read proc [port]) → unspecified

Streaming event-based JSON reader. PORT must be a textual input port. The default value of PORT is the value returned by the procedure current-input-port. The returned value is unspecified.

json-stream-read should raise an object that satisfies the predicate json-error? in cases where the JSON is invalid.

json-stream-read will call PROC with two arguments. The first argument is the event type, and the second argument is the event obj. Since the value that can be taken by obj depends on the value of type, they are presented together in the following paragraph.

The first arguments is denoted type. It is a symbol. It can be the following:

  • 'json-structure
  • 'json-value

The obj argument can take the following values depending on type:

  • If type is 'json-structure, then obj can be:

    • 'array-open symbol denoting that an array should be constructed.

    • 'array-close symbol denoting that the construction of the array for which the last 'array-open was emitted and not closed is finished.

    • 'object-open symbol denoting that an object should be constructed. The object's key-value pairs are emitted in sequence like those in a property list (plist). The keys will be produced by (proc 'json-structure some-symbol), that is, the key is always a symbol. The production of the key is always followed by the production of a value. Otherwise, the JSON would be invalid and json-stream-read would raise error. The production of the value can be a 'json-value but can also be 'json-structure with obj strictly equivalent 'array-open or 'object-open. These events will be followed by other events. Eventually, an associated 'object-close event is emitted.

    • 'object-close symbol denoting that the construction of the object for which the last object-open was emitted and not closed is finished.

  • If type is 'json-value, then obj is scheme object that can be of the following types:

    • symbol

    • number

    • string

Note: This document does not prescribe a validating parser. But in cases where the input JSON is valid, json-parse must emit the correct sequence of events.

Note: Given that type is 'json-value, there are two occasions where obj can be a symbol: 1) the JSON contains a null value, in that case 'null symbol is produced; 2) In a JSON object, keys are produced with the type 'json-value and the obj as a symbol. In particular the key "null" will be represented with 'null symbol and JSON null too, but that does not have bad consequences for the user.

Example One

Take for instance the following JSON text:

[true, false, 42, null]

It will result in the following calls to PROC:

(proc 'json-structure 'array-open)
(proc 'json-value #t)
(proc 'json-value #f)
(proc 'json-value 42)
(proc 'json-value 'null)
(proc 'json-structure 'array-close)

Example Two

Take for instance the following JSON text:

["hello", "world", {"truth": [42]}]

It will result in the following calls to PROC:

(proc 'json-structure 'array-open)
(proc 'json-value "hello")
(proc 'json-value "world")
(proc 'json-structure 'object-open)
(proc 'json-value 'truth)
(proc 'json-structure 'array-open)
(proc 'json-value 42)
(proc 'json-structure 'array-close)
(proc 'json-structure 'object-close)
(proc 'json-structure 'array-close)

(json-read [port]) → object

JSON reader procedure. PORT must be a textual input port. The default value of PORT is the value returned by the procedure current-input-port. The returned value is a scheme object.

The mapping between JSON types and Scheme objects is the following:

  • null → the symbol 'null
  • true → #t
  • false → #f
  • number → number
  • string → string
  • array → vector
  • object → association list with keys that are symbols

(json-write obj [port]) → unspecified

JSON writer procedure. PORT must be a textual output port. The default value of PORT is the value returned by the procedure current-output-port. The returned value is unspecified.

json-write will validate that OBJ can be serialized into JSON before writing to PORT. In the case where OBJ is not an object or composition of the following types, an error that satisfies json-error? is raised:

  • symbol 'null
  • boolean
  • number must be integers or inexact rational (that is they must not be complex, infinite, nan, or exact rational that are not integers)
  • string
  • vector
  • association list
